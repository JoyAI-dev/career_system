# Q-3: 问卷恢复与完成流程

## 概述
验证用户在部分完成问卷后重新登录的恢复流程：公告可立即关闭 → 偏好已完成自动跳过 → 问卷从草稿恢复 → 导航到最后话题 → 回答预留的最后一个问题 → 提交 → 自动跳转到 dashboard。

## 前置条件
- 执行 `seed.ts` 创建 a13 用户并预设：
  - 公告已查看（AnnouncementView 记录存在）
  - 偏好已完成（UserPreference + selections 记录存在）
  - 问卷草稿已保存（ResponseSnapshot isSnapshot=false + 大部分 ResponseAnswer），最后一个话题的最后一个问题故意留空
  - **没有** isSnapshot=true 的记录（尚未正式提交）

## 测试步骤

### Phase 1: 登录与公告验证

**Step 1.1: 登录**
- 导航到 `https://career-staging.joysort.cn/login`
- 输入用户名: `a13`，密码: `a13`
- 点击登录
- **检查点 CP-1**: 登录成功

**Step 1.2: 公告验证**
- 如果公告弹出，应可立即关闭
- **检查点 CP-2**: 公告无倒计时，关闭按钮立即可用
- 点击关闭按钮
- **检查点 CP-3**: 公告消失
- 📸 截图

### Phase 2: 问卷恢复验证

**Step 2.1: 确认进入问卷页面**
- 由于偏好已完成但问卷未提交，应自动进入问卷页面
- **检查点 CP-4**: 页面显示问卷界面（QuestionnaireFlow），URL 含 `/questionnaire`
- 📸 截图

**Step 2.2: 检查草稿恢复**
- 问卷应加载之前保存的草稿答案
- 在第一个话题中，检查问题是否已有选中的选项（radio button 已选中）
- **检查点 CP-5**: 第一个话题的问题有已选中的答案（草稿恢复成功）
- 📸 截图（显示已恢复的选中状态）

### Phase 3: 导航到最后话题并回答

**Step 3.1: 导航到最后一个话题 tab**
- 问卷共有 14 个话题 tab，点击最后一个（第14个："生育福利"）
- **检查点 CP-6**: 成功切换到最后一个话题
- 📸 截图

**Step 3.2: 检查最后一个问题的状态**
- 在最后一个话题中，找到最后一个维度的最后一个问题
- **检查点 CP-7**: 该问题没有选中的答案（是空的，未回答）
- 📸 截图（显示留空的问题）

**Step 3.3: 回答最后一个问题**
- 选择任意一个答案选项（如"做过信息搜索"）
- **检查点 CP-8**: 答案选项被选中
- 📸 截图

### Phase 4: 提交问卷

**Step 4.1: 提交**
- 在最后一个话题页面，应该能看到"提交"按钮（而非"下一个话题"）
- 点击提交按钮
- **检查点 CP-9**: 提交按钮可见
- 📸 截图（提交前状态）

**Step 4.2: 确认提交结果**
- 点击提交后，如果有确认对话框，点击确认
- **检查点 CP-10**: 提交成功，页面跳转

### Phase 5: Dashboard 验证

**Step 5.1: 确认到达 Dashboard**
- **检查点 CP-11**: URL 变为 `/dashboard` 或显示 dashboard 内容
- **检查点 CP-12**: Dashboard 页面正常渲染（显示活动步进器、活动浏览器等内容）
- 📸 截图（dashboard 完整页面）

**Step 5.2: 确认不再被重定向到问卷**
- 刷新页面
- **检查点 CP-13**: 仍在 dashboard 页面（不被重定向到 /questionnaire）
- 📸 截图

## 完成标准

| 检查点 | 描述 | 预期 |
|--------|------|------|
| CP-1 | 登录成功 | 页面跳转 |
| CP-2 | 公告无倒计时 | 可立即关闭 |
| CP-3 | 公告关闭 | 弹窗消失 |
| CP-4 | 进入问卷页 | URL 含 /questionnaire |
| CP-5 | 草稿恢复 | 问题有已选答案 |
| CP-6 | 切换到最后话题 | 第14个 tab 内容显示 |
| CP-7 | 最后问题留空 | 无选中答案 |
| CP-8 | 选中答案 | 选项被选中 |
| CP-9 | 提交按钮可见 | 最后话题显示提交按钮 |
| CP-10 | 提交成功 | 页面跳转 |
| CP-11 | 到达 Dashboard | URL=/dashboard |
| CP-12 | Dashboard 正常 | 页面内容正确 |
| CP-13 | 不再重定向 | 刷新仍在 dashboard |

## 关键验证逻辑
- 提交问卷的核心是创建 `ResponseSnapshot(isSnapshot=true)` 记录
- Dashboard 守门逻辑检查 `hasCompletedQuestionnaire()`，即是否存在 `isSnapshot=true` 的记录
- 如果草稿恢复失败（已答问题不显示为已选中），但提交功能仍正常，则 CP-5 失败但后续步骤可能仍通过
