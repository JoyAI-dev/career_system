# Q-1: 新用户公告流程

## 概述
验证新用户从登录到公告交互的完整流程，包括首次强制倒计时和重登录后的免倒计时行为。

## 前置条件
- 执行 `seed.ts` 创建干净的 a11 用户（无任何历史数据）
- staging 环境有至少一个激活的公告（由系统 seed 保证）

## 测试步骤

### Phase 1: 首次登录与公告倒计时

**Step 1.1: 登录**
- 导航到 `https://career-staging.joysort.cn/login`
- 输入用户名: `a11`，密码: `a11`
- 点击登录按钮
- **检查点 CP-1**: 登录成功，页面跳转（不停留在 login 页面）
- 📸 截图

**Step 1.2: 公告弹出（强制倒计时）**
- 登录后应看到公告弹窗（Dialog/Modal）
- **检查点 CP-2**: 公告弹窗可见
- **检查点 CP-3**: 倒计时数字可见（应显示 ≤20 的数字），且关闭按钮不可点击或不存在
- 📸 截图（捕获倒计时状态）

**Step 1.3: 等待倒计时结束**
- 等待 22 秒（20秒倒计时 + 2秒缓冲）
- **检查点 CP-4**: 倒计时消失或显示为0，关闭按钮变为可点击状态
- 📸 截图（捕获可关闭状态）

**Step 1.4: 关闭公告**
- 点击关闭按钮（或 "我知道了" 按钮）
- **检查点 CP-5**: 公告弹窗消失
- **检查点 CP-6**: 页面显示偏好选择界面（/preferences 页面内容）
- 📸 截图

### Phase 2: 确认进入偏好页面

**Step 2.1: 偏好页面验证**
- **检查点 CP-7**: 当前页面 URL 包含 `/preferences` 或页面显示偏好类别列表
- 确认可以看到偏好选择的 UI 元素（类别卡片/选项）
- 📸 截图

### Phase 3: 登出并重新登录

**Step 3.1: 登出**
- 在页面上找到登出按钮/链接（可能在顶部工具栏的 MiniToolbar 中）
- 点击登出
- **检查点 CP-8**: 返回到登录页面
- 📸 截图

**Step 3.2: 重新登录**
- 再次输入用户名: `a11`，密码: `a11`
- 点击登录
- **检查点 CP-9**: 登录成功

**Step 3.3: 公告弹出（无倒计时）**
- **检查点 CP-10**: 公告弹窗再次出现
- **检查点 CP-11**: 没有倒计时数字显示，关闭按钮立即可点击
- 📸 截图（关键验证点：无倒计时）

**Step 3.4: 立即关闭公告**
- 点击关闭按钮
- **检查点 CP-12**: 公告立即消失
- **检查点 CP-13**: 仍在偏好选择页面（因为用户尚未完成偏好）
- 📸 截图

## 完成标准

| 检查点 | 描述 | 预期 |
|--------|------|------|
| CP-1 | 登录成功 | 页面跳转离开 /login |
| CP-2 | 公告弹窗可见 | Dialog/Modal 出现 |
| CP-3 | 倒计时可见且不可关闭 | 显示秒数，无可点击的关闭按钮 |
| CP-4 | 倒计时结束可关闭 | 关闭按钮可点击 |
| CP-5 | 公告关闭 | 弹窗消失 |
| CP-6 | 进入偏好页 | 显示偏好选择内容 |
| CP-7 | 偏好页面正确 | URL 含 /preferences |
| CP-8 | 登出成功 | 回到登录页 |
| CP-9 | 重新登录成功 | 页面跳转 |
| CP-10 | 公告再次出现 | 弹窗可见 |
| CP-11 | 无倒计时 | 无秒数显示，可立即关闭 |
| CP-12 | 立即关闭 | 弹窗消失 |
| CP-13 | 仍在偏好页 | 显示偏好选择内容 |

## 已知限制
- agent-browser 的 sessionStorage 在新 session 中是空的，这意味着每次执行测试都等同于新浏览器窗口
- 如果 staging 没有激活的公告，CP-2 ~ CP-5 和 CP-10 ~ CP-12 会失败（这本身也是一个发现）
